This guide will help you go from zero to a fully configured development environment in just a few minutes.

## TL;DR

```bash
# Clone and install
git clone https://github.com/ombratteng/dotfiles.git ~/.dotfiles
cd ~/.dotfiles
./bootstrap.sh --force

# Restart your terminal and enjoy!
```

## Step-by-Step Quick Start

<Steps>
  <Step title="Clone the repository">
    Open your terminal and clone the dotfiles repository:
    
    ```bash
    git clone https://github.com/ombratteng/dotfiles.git ~/.dotfiles
    cd ~/.dotfiles
    ```
    
    This downloads the dotfiles to a hidden `.dotfiles` directory in your home folder.
  </Step>
  
  <Step title="Run the bootstrap script">
    Execute the bootstrap script with the `--force` flag for automatic installation:
    
    ```bash
    ./bootstrap.sh --force
    ```
    
    This will:
    - Install Oh My Zsh and Powerlevel10k theme
    - Install modern CLI tools (bat, eza, atuin, etc.)
    - Switch your default shell to Zsh
    - Copy all configuration files to your home directory
    
    <Note>
    The installation takes 2-5 minutes depending on your internet connection and system.
    </Note>
  </Step>
  
  <Step title="Restart your terminal">
    Close and reopen your terminal to apply all changes:
    
    ```bash
    exit
    ```
    
    Open a new terminal window. You should see the Powerlevel10k prompt.
  </Step>
  
  <Step title="Configure Powerlevel10k (First time only)">
    If this is your first time using Powerlevel10k, the configuration wizard will start automatically. Follow the prompts to customize your prompt style.
    
    If the wizard doesn't start, run:
    
    ```bash
    p10k configure
    ```
    
    <Tip>
    Choose the "Rainbow" style for a colorful, informative prompt, or experiment with different styles to find what you like.
    </Tip>
  </Step>
</Steps>

### Zsh Features

Your new shell includes powerful features:

<Steps>
  <Step title="Command autosuggestions">
    Start typing a command you've used before. Zsh will suggest the full command in gray text. Press `→` (right arrow) to accept.
    
    ```bash
    git st[→]  # Completes to 'git status'
    ```
  </Step>
  
  <Step title="Syntax highlighting">
    Valid commands are highlighted in green, invalid ones in red as you type.
  </Step>
  
  <Step title="Advanced tab completion">
    Press `Tab` to see intelligent completions for commands, arguments, and files.
    
    ```bash
    git che[Tab]  # Shows 'checkout', 'cherry', 'cherry-pick'
    ```
  </Step>
  
  <Step title="Command history search">
    Press `Ctrl+R` to search through your command history with atuin.
  </Step>
</Steps>

## Personalizing Your Setup

### Add Personal Git Configuration

Create a `.gitconfig.local` file for your personal Git settings:

```bash
cat > ~/.gitconfig.local << EOF
[user]
	name = Your Name
	email = your.email@example.com
[github]
	user = yourusername
EOF
```

### Add Custom Environment Variables

Add personal environment variables to `.zshenv.local`:

```bash
cat > ~/.zshenv.local << EOF
# Personal environment variables
export EDITOR="vim"
export PROJECTS_DIR="$HOME/projects"
export MY_API_KEY="your-secret-key"
EOF
```

### Add Custom Aliases and Functions

Add personal shell customizations to `.zshrc.local`:

```bash
cat > ~/.zshrc.local << EOF
# Personal aliases
alias myproject="cd ~/projects/my-project"
alias serve="python -m http.server 8000"

# Personal functions
function mkcd() {
    mkdir -p "$1" && cd "$1"
}
EOF
```

Then reload your configuration:

```bash
source ~/.zshrc
```

<Warning>
Never edit the main dotfiles directly (`.zshrc`, `.zshenv`, `.gitconfig`) as they will be overwritten when you sync or upgrade. Always use `.local` files for customizations.
</Warning>

## Common Workflows

### Daily Usage

```bash
# Start your day
cd ~/projects/my-project

# Work with Git (enhanced prompt shows branch and status)
git status          # Or just observe the prompt
git graph           # View commit history
git recent-branches # Switch to recent branch

# View files with syntax highlighting
cat src/main.js

# Search command history
# Press Ctrl+R and type to search
```

### Keeping Dotfiles Updated

```bash
# Update all tools to latest versions
cd ~/.dotfiles
./bootstrap.sh --upgrade

# Just sync configuration changes
./bootstrap.sh --sync
```

<Info>
Run `--upgrade` monthly to keep your tools and configurations current with the latest features and security updates.
</Info>

### Working Across Multiple Machines

```bash
# On machine 1: Make changes to dotfiles
cd ~/.dotfiles
# Edit files or add new configurations
git add .
git commit -m "Add custom configurations"
git push

# On machine 2: Pull and sync changes
cd ~/.dotfiles
./bootstrap.sh --sync
```

## Useful Zsh Shortcuts

Master these keyboard shortcuts to boost your productivity:

| Shortcut | Action |
|----------|--------|
| `Ctrl+R` | Search command history with atuin |
| `Ctrl+A` | Move cursor to beginning of line |
| `Ctrl+E` | Move cursor to end of line |
| `Ctrl+W` | Delete word before cursor |
| `Ctrl+U` | Clear line before cursor |
| `Ctrl+K` | Clear line after cursor |
| `Ctrl+L` | Clear screen (keep current line) |
| `Alt+.`  | Insert last argument from previous command |
| `!!`     | Repeat last command |
| `sudo !!`| Repeat last command with sudo |
| `→`      | Accept autosuggestion |

## Troubleshooting Quick Fixes

### Prompt looks broken

Missing font glyphs? Install a Nerd Font:

```bash
# macOS
brew tap homebrew/cask-fonts
brew install --cask font-meslo-lg-nerd-font

# Then configure your terminal to use "MesloLGS NF" font
```

### Commands not found

Reload your shell configuration:

```bash
source ~/.zshrc
```

Or restart your terminal.

### Git status not showing in prompt

Make sure you're in a Git repository:

```bash
cd ~/.dotfiles
# Now you should see Git info in your prompt
```

### Autosuggestions not working

The feature may be disabled. Check if the plugin is loaded:

```bash
echo $plugins | grep autosuggestions
```

If not found, the plugin may need to be reinstalled:

```bash
cd ~/.dotfiles
./bootstrap.sh --upgrade zsh_autosuggestions
```
