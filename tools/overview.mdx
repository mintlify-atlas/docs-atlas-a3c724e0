|-----------------|-------------------|------------------|

| `ls` | `eza` | Git status, icons, tree view, better formatting |
| Standard prompt | `powerlevel10k` | Git info, fast rendering, customizable segments |

## Command Aliases

The configuration automatically sets up convenient aliases for these tools:

### File Listing (Eza)

```bash
# Basic listing with eza
ls              # Replaces default ls

# Long format with git status
l               # eza --long --group --git --header --mounts

# Show all files including hidden
la              # eza --long --group --git --header --mounts --all

# Additional variants
laz             # la with SELinux context
lao             # la with octal permissions
```

### File Viewing (Bat)

```bash
# View file with syntax highlighting
cat filename.js  # Automatically uses bat

# Bat-specific features
bat --language=json config.json    # Force language
bat --show-all file.txt            # Show non-printable characters
bat --diff file1.txt file2.txt     # Show diff
```

### Shell History (Atuin)

```bash
# Search history (interactive)
Ctrl+R          # Opens Atuin search interface

# Atuin commands
atuin search <query>     # Search history
atuin stats              # View statistics
atuin sync               # Sync history across machines
```

## Installation Methods

### macOS (Homebrew)

```bash
# All tools can be installed via Homebrew
brew install atuin bat eza
```

### Linux

The dotfiles include automated installation scripts for each tool:

```bash
# Using the bootstrap script
./bootstrap.sh

# Or manually for specific tools
zsh scripts/atuin.zsh install
zsh scripts/bat.zsh install
zsh scripts/eza.zsh install
```

<Note>
On Linux, most tools are installed from official releases or system package managers. The scripts handle version checking and automatic updates.
</Note>

## Zsh Plugins

The configuration includes essential Zsh plugins that enhance the shell experience:

### Zsh Autosuggestions

Suggests commands as you type based on history and completions.

```bash
# Installed to
~/.oh-my-zsh/custom/plugins/zsh-autosuggestions

# Accept suggestion with
→  # Right arrow key
```

### Zsh Syntax Highlighting

Provides real-time syntax highlighting for commands.

```bash
# Installed to
~/.oh-my-zsh/custom/plugins/zsh-syntax-highlighting

# Features
- Valid commands: green
- Invalid commands: red
- Paths: underlined
- Strings: yellow
```

### Oh My Zsh Built-in Plugins

The configuration enables numerous useful plugins:

```bash
plugins=(
  aliases              # Alias management
  colored-man-pages    # Colorized man pages
  docker              # Docker completions
  docker-compose      # Docker Compose completions
  gh                  # GitHub CLI completions
  git                 # Git aliases and completions
  gitignore           # gitignore.io integration
  helm                # Helm completions
  kubectl             # Kubernetes completions
  kubectx             # kubectx/kubens completions
  pip                 # Python pip completions
  poetry              # Poetry completions
  sudo                # ESC ESC to prefix sudo
  yarn                # Yarn completions
  zsh-autosuggestions
  zsh-syntax-highlighting
)
```

<Tip>
Press `ESC` twice to add `sudo` to the beginning of the current command - a handy feature from the sudo plugin!
</Tip>

## Configuration Files

### Main Configuration

The primary Zsh configuration is in `.zshrc`:

```bash
# Theme
ZSH_THEME="powerlevel10k/powerlevel10k"

# History format
HIST_STAMPS="dd.mm.yyyy"

# Tool initialization
eval "$(atuin init zsh --disable-up-arrow)"
```

### Environment Variables

Disable specific tools if needed:

```bash
# In .zshrc.local or environment
export _DISABLE_ATUIN=1        # Disable Atuin
export _DISABLE_BAT=1          # Disable Bat
export _DISABLE_EZA=1          # Disable Eza
export _DISABLE_NEOFETCH=1     # Disable Neofetch
```

## Updating Tools

### Via Bootstrap Script

```bash
# Update all tools
./bootstrap.sh
```

### Individual Tool Updates

```bash
# Update specific tools
zsh scripts/atuin.zsh upgrade
zsh scripts/bat.zsh upgrade
zsh scripts/eza.zsh upgrade
zsh scripts/powerlevel10k.zsh upgrade
zsh scripts/oh_my_zsh.zsh upgrade
```

<Info>
On macOS, upgrades are handled by Homebrew. On Linux, the scripts check GitHub releases for the latest versions and install them automatically.
</Info>

## Additional Tools

### LS_COLORS

Enhanced color scheme for file listings from [trapd00r/LS_COLORS](https://github.com/trapd00r/LS_COLORS):

```bash
# Installed to
${XDG_CONFIG_HOME}/lscolors/lscolors.sh

# Loaded automatically in .zshrc
. "${XDG_CONFIG_HOME}/lscolors/lscolors.sh"
```

The LS_COLORS project provides a comprehensive color configuration for directories, symlinks, executables, archives, images, videos, and more. It's automatically installed and configured by the bootstrap script.

### nanorc

Syntax highlighting for nano editor from [scopatz/nanorc](https://github.com/scopatz/nanorc):

```bash
# Installed to
~/.nano/

# Provides syntax files for:
# - Programming languages (Python, JavaScript, Go, etc.)
# - Configuration files (YAML, JSON, INI, etc.)
# - Markup languages (Markdown, HTML, XML, etc.)
# - And many more
```

Enhances the nano editor experience with colorful syntax highlighting for numerous file types.

### macOS-Specific Tools

The following tools are installed only on macOS systems:

#### kubectl

Kubernetes command-line tool for cluster management:

```bash
# Installed via Homebrew
brew install kubernetes-cli

# Provides kubectl command for Kubernetes operations
kubectl get pods
kubectl apply -f deployment.yaml
```

#### kubectx

Fast way to switch between Kubernetes contexts and namespaces:

```bash
# Installed via Homebrew
brew install kubectx

# Switch contexts quickly
kubectx production
kubectx staging

# Switch namespaces
kubens default
kubens kube-system
```

The dotfiles include aliases for quick access:
```bash
alias kctx='kubectx'
alias kns='kubens'
```

#### n

Node.js version manager:

```bash
# Installed via Homebrew
brew install n

# Configured in .zshenv.local
export N_PREFIX="$HOME/.config/n"
export PATH="$HOME/.config/n/bin:$PATH"

# Usage
n latest          # Install latest Node.js version
n lts             # Install latest LTS version
n 18.17.0         # Install specific version
n                 # Show installed versions and select
```

<Note>
These tools are only installed on macOS because the bootstrap script detects the platform and conditionally adds them to the installation list. On Linux systems, Kubernetes tools are typically installed via the system package manager or cloud provider CLIs.
</Note>

### Neofetch

Displays system information on shell startup:

```bash
# Automatically runs if installed
# Can be disabled with
export _DISABLE_NEOFETCH=1
```

### Completion System

Enhanced completion system with multiple sources:

```bash
# Zsh completions
autoload -Uz compinit && compinit -u

# Bash completions compatibility
autoload -U bashcompinit && bashcompinit
```
