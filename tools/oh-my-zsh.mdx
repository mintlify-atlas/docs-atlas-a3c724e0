## Installation

```bash
# Using the dotfiles bootstrap script
./bootstrap.sh

# Or install manually
zsh scripts/oh_my_zsh.zsh install
```

The installation script clones Oh My Zsh into your home directory:

```bash
# Installation location
~/.oh-my-zsh
```

<Info>
Oh My Zsh requires Zsh 5.0.8 or more recent. On macOS, Zsh is the default shell. On Linux, install it via your package manager: `apt install zsh` or `dnf install zsh`.
</Info>

## Configuration

### Main Configuration File

Oh My Zsh is configured in `.zshrc`:

```bash
# Theme selection
ZSH_THEME="powerlevel10k/powerlevel10k"

# History timestamp format
HIST_STAMPS="dd.mm.yyyy"

# Enable plugins
plugins=(
  aliases
  colored-man-pages
  docker
  docker-compose
  gh
  git
  gitignore
  helm
  kubectl
  kubectx
  pip
  poetry
  projects
  sublime
  sudo
  yarn
  zsh-autosuggestions
  zsh-completions
  zsh-syntax-highlighting
)

# Initialize Oh My Zsh
source "${ZSH}"/oh-my-zsh.sh
```

### Environment Variable

The `$ZSH` environment variable points to the Oh My Zsh directory:

```bash
# Set in .zshenv or automatically by Oh My Zsh
export ZSH="${HOME}/.oh-my-zsh"
```

## Customization

### Local Configuration

Create `.zshrc.local` for machine-specific settings:

```bash
# .zshrc.local - Loaded at the end of .zshrc
export CUSTOM_VAR="value"

# Custom aliases
alias myproject="cd ~/projects/myproject"

# Environment-specific settings
export AWS_PROFILE="dev"
```

<Info>
The dotfiles automatically source `.zshrc.local` if it exists, allowing you to keep machine-specific configuration separate from the main config.
</Info>

### Custom Aliases

Create custom aliases for your workflow:

```bash
# In .zshrc.local or .zshrc

# Quick navigation
alias projects="cd ~/projects"
alias dotfiles="cd ~/dotfiles"

# Git shortcuts
alias gs="git status"
alias gpl="git pull"
alias gps="git push"

# Docker shortcuts
alias dcu="docker-compose up"
alias dcd="docker-compose down"

# Custom commands
alias update-all="brew update && brew upgrade && omz update"
```

### Custom Functions

Add helper functions:

```bash
# In .zshrc.local

# Create and enter directory
mkcd() {
  mkdir -p "$1" && cd "$1"
}

# Quick Git commit
gquick() {
  git add . && git commit -m "$1" && git push
}

# Extract archives
extract() {
  if [ -f $1 ]; then
    case $1 in
      *.tar.bz2) tar xjf $1 ;;
      *.tar.gz) tar xzf $1 ;;
      *.zip) unzip $1 ;;
      *.7z) 7z x $1 ;;
      *) echo "'$1' cannot be extracted" ;;
    esac
  fi
}
```

## Additional Configuration

### History Settings

Configure command history:

```bash
# In .zshrc

# History file location
HISTFILE=~/.zsh_history

# History size
HISTSIZE=10000
SAVEHIST=10000

# History options
setopt HIST_IGNORE_ALL_DUPS  # Remove duplicate entries
setopt HIST_FIND_NO_DUPS     # Don't display duplicates
setopt HIST_IGNORE_SPACE     # Don't save commands starting with space
setopt SHARE_HISTORY         # Share history between sessions
```

### Completion Settings

Enhanced completion system:

```bash
# In .zshrc

# Enable completions
autoload -Uz compinit && compinit -u
autoload -U bashcompinit && bashcompinit

# Completion options
zstyle ':completion:*' menu select                    # Interactive menu
zstyle ':completion:*' matcher-list 'm:{a-z}={A-Z}'  # Case insensitive
zstyle ':completion:*' list-colors ${(s.:.)LS_COLORS} # Colored completions
```

### Additional Snippets

Load additional configuration from a snippets directory:

```bash
# In .zshrc (included in dotfiles)
for zshrc_snipplet in $XDG_CONFIG_HOME/zsh.d/S[0-9][0-9]*[^~] ; do
    source "${zshrc_snipplet}"
done
```

This loads all files matching the pattern from `~/.config/zsh.d/`:

```bash
# Create snippets directory
mkdir -p ~/.config/zsh.d

# Add numbered snippets
~/.config/zsh.d/S10-custom-exports.zsh
~/.config/zsh.d/S20-custom-aliases.zsh
~/.config/zsh.d/S30-custom-functions.zsh
```

## Updating

### Manual Update

```bash
# Update Oh My Zsh
omz update

# Force update
omz update --unattended
```

### Using Update Script

```bash
# Update via dotfiles script
zsh scripts/oh_my_zsh.zsh upgrade
```

<Note>
Oh My Zsh automatically checks for updates every ~2 weeks and prompts you to update.
</Note>

### Disable Auto-Update Prompts

```bash
# In .zshrc
zstyle ':omz:update' mode disabled
```

## Plugin Management

### Enabling Plugins

Add to the `plugins` array in `.zshrc`:

```bash
plugins=(
  git
  docker
  # ... other plugins
  new-plugin  # Add your plugin here
)
```

### Installing Custom Plugins

```bash
# Clone to custom plugins directory
git clone <plugin-repo> ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/<plugin-name>

# Enable in .zshrc
plugins=(... <plugin-name>)

# Reload
source ~/.zshrc
```

### Disabling Plugins

Remove from the `plugins` array:

```bash
plugins=(
  git
  docker
  # kubectl  # Commented out - disabled
)
```

## Troubleshooting

### Slow Shell Startup

Profile your Zsh startup:

```bash
# Benchmark startup time
time zsh -i -c exit

# Enable profiling
zmodload zsh/zprof
# ... at top of .zshrc

# Run profiling report at end of .zshrc
zprof
```

Common causes:

- Too many plugins enabled
- Slow plugins (NVM, RVM, etc.)
- Large completion system

Solutions:

```bash
# Lazy-load slow tools
function nvm() {
  unset -f nvm
  source ~/.nvm/nvm.sh
  nvm "$@"
}

# Reduce plugin count
# Disable unused completions
```

### Plugin Not Working

Check if the plugin is:

1. Listed in the `plugins` array
2. Installed (for custom plugins)
3. Compatible with your Zsh version

```bash
# Verify plugin is enabled
echo $plugins

# Check plugin exists
ls ${ZSH}/plugins/<plugin-name>
# or for custom
ls ${ZSH_CUSTOM}/plugins/<plugin-name>
```

### Completion Not Working

Rebuild completion cache:

```bash
# Remove completion cache
rm ~/.zcompdump*

# Reload Zsh
source ~/.zshrc

# Completions should rebuild automatically
```

## Tips and Best Practices

<Tip>
Keep your `.zshrc` organized by moving machine-specific configurations to `.zshrc.local` and using the `~/.config/zsh.d/` snippets directory for modular configuration.
</Tip>

### Performance Optimization

```bash
# Load plugins selectively based on available commands
(( $+commands[docker] )) && plugins+=(docker)
(( $+commands[kubectl] )) && plugins+=(kubectl)

# Skip aliases you don't use
zstyle ':omz:plugins:git' aliases no
```

### Backup Configuration

```bash
# Backup before making changes
cp ~/.zshrc ~/.zshrc.backup

# Version control your dotfiles
cd ~ && git init
git add .zshrc .zshrc.local
git commit -m "Backup Zsh config"
```

### Explore New Plugins

```bash
# Browse available plugins
ls ${ZSH}/plugins/

# Read plugin documentation
cat ${ZSH}/plugins/<plugin-name>/README.md

# Try plugins temporarily (without editing .zshrc)
source ${ZSH}/plugins/<plugin-name>/<plugin-name>.plugin.zsh
```
